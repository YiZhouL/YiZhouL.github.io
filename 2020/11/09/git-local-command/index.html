Git设置

git通过不同的设置层次级别实现定制化。

在git有四种设置级别：

  1. system：系统级别，配置文件位于 /etc/gitconfig
  2. user：用户级别，配置文件位于 ~/.gitconfig (旧版本位于~/git/config)
  3. repository: 仓库级别，配置文件位于 .git/config

一个git仓库最终的配置项将会从上到下不断更新(重复将覆盖)的结果。

所有的git操作都从新建一个仓库开始

├── hooks/ # 一些关键git操作触发的钩子
│ ├── applypatch-msg.sample

│ └── update.sample

│ └── exclude # 存放

│ ├── info
│ └── pack

├── heads/ # 分支引用
└── tags/ # 标签引用

git add file1 file2 … # 将指定文件更新
git add . # 递归将当前目录下所有文件更新
git add -A # 将仓库所有文件更新无论在那个目录下

git支持 * ？ [] 通配符

git commit -m “some commit message” # 提交时携带一些提交信息
git commit -a -m “message” # 自动暂存然后提交
git commit file1 file2 file3 … # git允许提交指定文件；需要注意如果之前INDEX已经更新过的那些文件是不会生效的，这次只会提交指定的文件；同时这些文件须满足已在git的追踪中

git restore file # restore命令默认会把file恢复到INDEX中的状态

git restore -s HEAD~2 file1 # [-s|–source]指定源

git restore [-S|–staged] file1 # 当加上–staged参数时，将会恢复INDEX中指定的文件，默认源使用HEAD

git restore -SW file1

git reset [–soft|–mixed|–hard] commitIndex | 根据不同级别重置

git rm –cached file1 file2 … # 从当前INDEX移除，但仍保留在工作目录

git rm file1 file2 … # 当前INDEX包含file1且工作目录中文件没有发生改变，从工作目录删除file1
git rm -f file1 file2 … # 同上，只不过当file1改变时需指定-f强制删除
git rm -r dir1 dir2 … # 递归

git mv [-f] # -f强制移动，如果destination已存在
git mv [-f] …

git status -s # 简短信息
git status -b # 当前所在分支

git branch [-l|–list] # 列出分支
git branch -r # 列出远程分支
git branch -a # 列出所有分支
git branch -vv # 查看各分支详细信息，实用
git branch new_branch # 从start-point创建新分支，start-point需要有一个commit对象，因此可以是HEAD、commit、tag，默认为HEAD；此时tree和index是否变化没有关系，只要没有提交所有分支共享tree和index直到有一个分支进行了提交，那么其余分支将会reset –hard 爱了爱了
git branch -u # 绑定远程分支，当你从绑定了远程分支创建分支时，并不会自动绑定
git branch –unset-stream # 取消绑定
git branch -f branch_name # 强制创建branch_name，不管存不存在，存在就覆写
git branch -d branch_name # 删除branch_name
git branch -m branch_new_name #　换名

git switch branch_name # 切换分支，分支间如果是父子关系不分叉则work tree不会改变。
git switch -f branch_name # 将修改忽略后，强制切换，切换后，以前分支修改没了

git merge b # 将b合并到HEAD
git merge -no-ff # ff合并时创建新的commit
git merge –abort # 只有合并发生冲突后才能使用，回到合并前的状态

git diff # 比较work tree和INDEX的文件变化
git diff –cached|–staged # 比较INDEX和HEAD的文件变化
git diff commit_index # 比较work tree 和某一commit_index的文件变化

git diff branch1 branch2 # 比较两分支之间的变化
git diff branch1 branch2 –stat # 只显示有差异的文件

git log –graph # 图形式
git log –oneline # 简短信息
git log branch_name # 某分支

git tag tag_name [commit] # 创建轻量标签
git tag [-l] # 列出标签
git tag -a tag_name [-m|-F] message [commit] # 创建附注标签
git tag -d tag_name # 删除标签
git tag -f # 强制创建，存在就覆写

git stash # 将change压入栈
git stash list # 列出
git stash pop [stash@{0}] # 将change弹出后应用在当前工作区
git stash apply [stash@{0}] # 只应用
git stash drop [stash@{0}] # 丢弃
git stash clear # 清理栈

git worktree list # 列出工作区信息
git worktree add commit # 添加一个工作区
git worktree remove [-f] # 移除一个工作区
git worktree prune # 清理手动删除工作目录的工作区



#### 总结
git对于同一功能实现有很多不同的命令(或组合)可以实现

我一直觉得掌握一门工具包括三点：
1. 工具的设计思想
2. 工具的使用规范
3. 一种合理的实践组合

对于特别复杂的工具比如git尤其需要做到；学习使用工具并不是最终目的。
